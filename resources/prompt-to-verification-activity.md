# Prompt-to-Verification Activity | Activité consigne-vérification

[English](#english) | [Français](#francais)

<a id="english"></a>

## English

Use case: help participants or students avoid accepting AI output without inspection.

Recommended duration: 10 to 20 minutes.

### Activity

1. Choose a topic you know well.

2. Ask an AI tool for a short teaching or explanation draft.

Starter prompt:

```text
I am preparing a short classroom activity about [topic] for students at [level].
Create a first draft with:
1. a short context,
2. two student-facing questions,
3. one common misconception,
4. three assumptions or claims that should be checked before use.
```

3. Review the output with this grid.

| Question | Notes |
|---|---|
| What is useful? |  |
| What is wrong, unclear, or too generic? |  |
| What must be verified before reuse? |  |
| What should be removed for privacy or accuracy? |  |

4. Decide whether the draft is:

- ready to adapt;
- useful only as inspiration;
- not worth keeping.

### Variation

Ask the same prompt in two tools, for example a usual AI tool and a course-specific assistant. Compare how each tool handles context, limits, verification, and privacy.

### Important Check

The point is not to prove that one tool is better. The point is to practice moving from output to expert review.

## Français

Cas d'usage: aider les personnes participantes ou les étudiants à ne pas accepter une sortie d'IA sans inspection.

Durée recommandée: 10 à 20 minutes.

### Activité

1. Choisir un sujet que vous connaissez bien.

2. Demander à un outil d'IA de produire un court brouillon pédagogique ou explicatif.

Consigne de départ:

```text
Je prépare une courte activité en classe sur [sujet] pour des étudiants de niveau [niveau].
Crée un premier brouillon avec:
1. un court contexte,
2. deux questions destinées aux étudiants,
3. une idée fausse fréquente,
4. trois hypothèses ou affirmations à vérifier avant utilisation.
```

3. Réviser la sortie avec cette grille.

| Question | Notes |
|---|---|
| Qu'est-ce qui est utile? |  |
| Qu'est-ce qui est faux, flou ou trop générique? |  |
| Qu'est-ce qui doit être vérifié avant réutilisation? |  |
| Qu'est-ce qui devrait être retiré pour des raisons de confidentialité ou d'exactitude? |  |

4. Décider si le brouillon est:

- prêt à adapter;
- utile seulement comme inspiration;
- non pertinent à garder.

### Variante

Poser la même consigne dans deux outils, par exemple un outil d'IA habituel et une IA personnalisée de cours. Comparer la façon dont chaque outil traite le contexte, les limites, la vérification et la confidentialité.

### Vérification importante

L'objectif n'est pas de prouver qu'un outil est meilleur. L'objectif est de pratiquer le passage d'une sortie brute vers une révision experte.
