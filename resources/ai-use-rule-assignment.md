# AI-Use Rule for a Take-Home Assignment | Règle d'usage de l'IA pour un devoir maison

[English](#english) | [Français](#francais)

<a id="english"></a>

## English

Use case: define a simple AI-use rule for a take-home data analysis assignment.

### Template Rule

```text
For this assignment, you may use AI tools for planning, debugging, and improving clarity.

You may use AI to:
- brainstorm an analysis plan;
- ask for explanations of concepts already covered in the course;
- debug code when you understand the goal of the code;
- improve wording, structure, or clarity;
- generate a checklist of assumptions to verify.

You may not use AI to:
- produce a complete final answer for submission;
- fabricate data, references, results, or interpretations;
- bypass the required reasoning steps;
- enter identifiable student data, confidential data, or sensitive information.

You must submit a short AI-use statement:
1. Which tool did you use?
2. What did you use it for?
3. What did you change after reviewing the output?
4. What did you verify yourself?
```

### Class-Size Adjustment

Small group:

- Ask for a short reflection or oral check.
- Review a few AI-use statements in detail.

Medium group:

- Use a short structured disclosure form.
- Sample a subset for closer review.

Large group:

- Use a common rule and a common disclosure template.
- Grade the observable learning objective rather than trying to police every interaction.

### Important Check

Make sure the rule matches the learning objective. If the objective is independent coding, the rule should be stricter. If the objective is interpretation, communication, or critique, the rule may allow more AI support while keeping verification explicit.

## Français

Cas d'usage: définir une règle simple d'usage de l'IA pour un devoir maison d'analyse de données.

### Modèle de règle

```text
Pour ce devoir, vous pouvez utiliser des outils d'IA pour planifier, déboguer et améliorer la clarté.

Vous pouvez utiliser l'IA pour:
- réfléchir à un plan d'analyse;
- demander des explications sur des concepts déjà vus dans le cours;
- déboguer du code lorsque vous comprenez l'objectif du code;
- améliorer la formulation, la structure ou la clarté;
- générer une liste d'hypothèses à vérifier.

Vous ne pouvez pas utiliser l'IA pour:
- produire une réponse finale complète à remettre;
- inventer des données, références, résultats ou interprétations;
- contourner les étapes de raisonnement exigées;
- saisir des données étudiantes identifiables, des données confidentielles ou des informations sensibles.

Vous devez remettre une courte déclaration d'usage de l'IA:
1. Quel outil avez-vous utilisé?
2. Pour quoi l'avez-vous utilisé?
3. Qu'avez-vous changé après avoir révisé la sortie?
4. Qu'avez-vous vérifié vous-même?
```

### Adaptation selon la taille du groupe

Petit groupe:

- Demander une courte réflexion ou une vérification orale.
- Réviser quelques déclarations d'usage de l'IA en détail.

Groupe moyen:

- Utiliser un formulaire de déclaration court et structuré.
- Échantillonner une partie des travaux pour une révision plus fine.

Grand groupe:

- Utiliser une règle commune et un modèle commun de déclaration.
- Évaluer l'objectif d'apprentissage observable plutôt que tenter de surveiller chaque interaction.

### Vérification importante

S'assurer que la règle correspond à l'objectif d'apprentissage. Si l'objectif est le codage autonome, la règle devrait être plus stricte. Si l'objectif est l'interprétation, la communication ou la critique, la règle peut autoriser davantage d'aide par IA tout en gardant la vérification explicite.
