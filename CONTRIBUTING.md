## English

Thank you for considering a contribution to the AI in Teaching Gallery.

The gallery is meant to share useful, inspectable examples from the workshop. A contribution can be small and unfinished. It should help another instructor understand what was tried, what needs checking, and how the idea might be reused.

### Fastest Option

Use the GitHub form:

- [Propose a gallery contribution](https://github.com/AurelienNicosiaULaval/ssc-2026-ai-teaching-gallery/issues/new?template=gallery-contribution.yml)

The form asks for:

- title
- contributor name or group name, optional
- course or context
- teaching need
- tool used
- prompt or approach
- object produced
- important check or correction
- reuse possibility
- next step
- sharing consent

### Privacy Rules

Do not include:

- identifiable student data
- confidential institutional data
- protected course, research, or workplace material
- proprietary material
- sensitive personal information

Use general descriptions, anonymized examples, or synthetic examples.

### Pull Request Option

If you are comfortable with GitHub, you can also open a pull request that edits `assets/gallery.js`.

Please keep each contribution bilingual when possible. If you can only contribute in one language, that is acceptable. The facilitators can help adapt the other language.

## Français

Merci d'envisager une contribution à la Galerie IA et enseignement.

La galerie sert à partager des exemples utiles et inspectables issus de l'atelier. Une contribution peut être courte et inachevée. Elle devrait aider une autre personne enseignante à comprendre ce qui a été essayé, ce qui doit être vérifié et comment l'idée pourrait être réutilisée.

### Option la plus simple

Utilisez le formulaire GitHub:

- [Proposer une contribution à la galerie](https://github.com/AurelienNicosiaULaval/ssc-2026-ai-teaching-gallery/issues/new?template=gallery-contribution.yml)

Le formulaire demande:

- titre
- nom ou nom du groupe, optionnel
- cours ou contexte
- besoin pédagogique
- outil utilisé
- consigne ou approche
- objet produit
- vérification ou correction importante
- possibilité de réutilisation
- prochaine étape
- consentement au partage

### Règles de confidentialité

Ne pas inclure:

- données étudiantes identifiables
- données institutionnelles confidentielles
- matériel protégé de cours, de recherche ou de travail
- matériel propriétaire
- information personnelle sensible

Utilisez des descriptions générales, des exemples anonymisés ou des exemples synthétiques.

### Option pull request

Si vous êtes à l'aise avec GitHub, vous pouvez aussi ouvrir une pull request qui modifie `assets/gallery.js`.

Merci de garder chaque contribution bilingue lorsque possible. Si vous pouvez contribuer seulement dans une langue, c'est acceptable. Les facilitateurs pourront aider à adapter l'autre langue.
